Rigidbody2D

1) Body Type
* Dynamic: Rigidbody перебуває під впливом фізики (сил, зіткнень, гравітації).
* Kinematic: Рух контролюється безпосередньо через зміну transform, фізичні сили не впливають на об’єкт, але він може взаємодіяти з іншими Dynamic-об’єктами при зіткненнях.
* Static: Об’єкт нерухомий і не реагує на фізику, але інші об’єкти все ще можуть з ним стикатися.


2) Material (Physics Material 2D)
Дозволяє задати Physics Material 2D для налаштування коефіцієнтів тертя і пружності (bounce). Це впливає на те, наскільки об’єкт «ковзатиме» чи «відштовхуватиметься» при зіткненнях.


3) Simulated
Вмикає/вимикає фізичну симуляцію для цього Rigidbody. Якщо зняти галочку, об’єкт ігнорується фізичним рушієм.


4) Use Auto Mass
Якщо увімкнено, маса об’єкта автоматично розраховується на основі обсягу (площі) колайдера і щільності (Density) в Collider 2D. Якщо вимкнено, потрібно вручну вказати масу в полі Mass.


5) Mass
Ручне задання маси об’єкта (використовується лише тоді, коли Use Auto Mass вимкнено).


6) Linear Drag
«Лінійний опір». Визначає, наскільки швидко втрачається лінійна швидкість об’єкта (його рух уповільнюється). Чим більше значення, тим швидше об’єкт «гальмує» у просторі.


7) Angular Drag
«Кутовий опір». Визначає, наскільки швидко зменшується кутова швидкість об’єкта (швидкість обертання). Чим більше значення, тим швидше об’єкт припиняє обертатися.


8) Gravity Scale
Множник сили гравітації, яка впливає на об’єкт. Наприклад, якщо Gravity Scale = 0, об’єкт взагалі не відчуватиме гравітацію; якщо поставити 2, дія гравітації буде вдвічі сильнішою за стандартну.


9) Collision Detection
* Discrete: Стандартний тип визначення зіткнень, менш ресурсомісткий, але може «пропускати» швидкі, дрібні об’єкти.
* Continuous: Більш точний тип, зменшує «проліт» об’єктів один через одного на високій швидкості, але потребує більше ресурсів.
* Continuous Speculative (у деяких версіях Unity): спроба покращити Continuous, але все одно потребує тестування для конкретних сценаріїв.


10) Sleeping Mode
Визначає, як Rigidbody «засинає» (припиняє обчислення фізики) при відсутності руху/взаємодій.
* Never Sleep: ніколи не переходить у режим сну.
* Start Awake: спочатку активний, потім може заснути, якщо відсутній рух.
* Start Asleep: спочатку «спить», прокидається від зіткнень чи застосування сил.


11) Interpolate
* None: без інтерполяції, переміщення може бути рваним під час оновлень.
* Interpolate: згладжує рух об’єкта між фізичними оновленнями (корисно для плавнішої анімації).
* Extrapolate: подібно до Interpolate, але «прогнозує» майбутній стан, що може давати плавніший рух, але іноді викликає позиційні похибки.

12) Constraints (у вкладці Constraints)
Можна «заморозити» (Freeze) певні осі руху та обертання, щоб об’єкт не рухався/не обертався в небажаних напрямках. У 2D найчастіше заморожують обертання навколо Z, щоб об’єкт залишався на місці і не обертався під час зіткнень.

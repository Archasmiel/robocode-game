Ізометрія в Unity

Згадуємо, як працює TileMap в Unity. 
Вивчаємо поняття Ізометрії в іграх. 
Налаштовуємо правильне сортування рендеринга для реалізації ілюзії ізометрії. 
Використовуючи TileMap створюємо ігровий рівень.

Діти мають знати: 
Що таке Tilemap та Ізометрія. 
Які є інструменти для малювання Tilemap. 
Навіщо потрібен Transparency Sort Axis. 

Вміти: 
Створювати палітру для Tilemap. 
Малювати рівень.



1) розповісти дітям що таке Tilemap та Ізометрія
2) зайти в налаштування і поміняти сортування блоків не по вісі Z, а по вісі Y
    Edit -> Project Settings -> Graphics -> Transparency Sort Axis (x=0, y=1, z=0)
3) поміняти спрайтам з папки з матеріалами параметри 
    pixel_per_unit=64 filter_mode=Point
4) додати на сцену тайлмап2д з ізометрією
    LMB -> 2D Object -> Tilemap (Isometric)
5) відкрити вікно з палеткою
    Window -> 2D -> Tile Palette
6) створити нову палетку, зберегти її в папці Palettes
    додати всі блоки з папки Raised, зберегти в Palettes -> Tiles
    іноді може бути не-Sprite в папці, дивіться на файли
7) пояснити про інструменти в палетці (всі як в пейнт)
    важливо: палетка є файлом і ви можете її налаштувати в інспекторі
    щоб блоки були зручно показані їх можна підтягнути трохи вгору 
    щоб було так, в тайлмап2д Tile Anchor = (1, 1, 0)
    додаємо таке ж і до палетки, але краще її зробити Rectangle та збільшити Cell Gap
    бо багато дітей скаржилися що вибирати блоки незручно і невидно в ізометрії палетки
8) розказати що можна робити верхні рівні для карти, просто додавши тайлмапи
    щоб накладалися правильно є параметр Order In Layer, чим більше тим вище
    до кожного тайлмапа поміняти Tilemap Renderer -> Mode -> Individual
    тоді блоки будуть накладатися правильно, чанки використовують коли є така система у грі
    у даному проекті всі блоки одночасно завантажені
9) створити нову палетку з спрайтів папки Colliders -> ColliderSprites
10) створити останній Tilemap 2D (Isometric) - у всіх налаштування однакові
    лише в ньому поміняти колір блоків з білого на червоний, матеріал не обов'язково
11) додати до цього тайлмап 
    TilemapCollider2D (Used By Composite = true)
    Rigidbody2D (Body Type = Static)
    CompositeCollider2D (не міняти)
12) додати програму для ховання тайлмапу з червоними блоками
    повісити на тайлмап з колайдерами
    на сцені бачимо наші колайдери, а у грі ні
    щоб не бачити в сцені біля об'єкту тайлмапу коли навестися є око та палець
    око не дає бачити цей об'єкт, але тільки на сцені - у грі видно
    палець не дає можливості випадково натискати, тобто прибирає виділення
    краще поставити на всі тайлмапи палець(перекреслений),
    а якщо треба на колайдерах ще й око(перекреслене)



програму можна спростити до такого 

public class HideColliderTilemap : MonoBehaviour {
    void Start() {
        GetComponent<TilemapRenderer>().enabled = false;
    }
}
